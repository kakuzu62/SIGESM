# AR-001 - Architecture Review da STS-001A

## Status

APROVADA COM RESSALVAS - CORRECAO OBRIGATORIA APLICADA EM AR-001R1.

## Checklist

- [ ] A funcionalidade esta em vertical slice.
- [ ] A View nao acessa SQLAlchemy.
- [ ] A View nao acessa repositories.
- [ ] A ViewModel nao contem regra de negocio.
- [ ] A Application orquestra a query.
- [ ] O dominio nao depende de Presentation.
- [ ] Infrastructure implementa contratos.
- [ ] DTO de listagem nao expoe senha, hash ou token.
- [ ] Paginacao evita carregar todos os registros na UI.
- [ ] O modulo foi integrado via NavigationService.
- [ ] Nao ha acoplamento indevido com outros modulos.

## Resultado da Revisao

AR-001 confirmou que o fluxo central da listagem segue a arquitetura prevista:

```text
UserListView
  -> UserListViewModel
  -> UserListingService
  -> ListUsersHandler
  -> IUserListingRepository
```

Nao foi encontrado acesso direto da View ao SQLAlchemy, nem acesso direto da
View a repositories. O DTO de listagem nao expoe senha, hash ou token.

## Correcoes AR-001R1

- Substituidas propriedades transitorias de abertura de dialogo por sinais
  explicitos `new_user_requested` e `edit_user_requested`.
- Adicionada propriedade `is_loading` na ViewModel para preparar o fluxo para
  persistencia real e futura execucao assincrona.
- Padronizadas notificacoes apos carregamento bem-sucedido para `users`, `page`,
  `total`, `total_pages` e `error_message`.
- Em falha, a politica adotada e preservar os dados previamente carregados e
  atualizar apenas `error_message`.
- Expandida a cobertura de testes para validacao, limites, resultado vazio,
  ultima pagina parcial, eventos de dialogo, erro e estado de carregamento.
- Removida dependencia de hash real em testes de listagem, usando hash Argon2id
  fixo valido apenas como dado de teste.

## Decisao Arquitetural

STS-001A introduz o primeiro vertical slice funcional menor. O escopo e
intencionalmente restrito a leitura, deixando comandos de escrita para STSs
posteriores.

## Debito Arquitetural Registrado

AR-001-01 permanece registrado: o slice `user_management` esta fisicamente em
`src/presentation/modules/user_management/`, apesar de conter subcamadas de
application, domain e infrastructure. A realocacao para `src/modules/` ou
`src/contexts/administration/` devera ser feita em refatoracao controlada antes
do crescimento do modulo.

## Riscos

- O adapter em memoria e adequado para bootstrap local, mas a persistencia real
  devera ser ativada em etapa posterior.
- Pesquisa incremental ainda nao possui debounce; a estrutura foi preparada para
  adiciona-lo sem alterar o fluxo Application.
- A comprovacao final depende da execucao completa do Quality Gate apos as
  correcoes AR-001R1.
