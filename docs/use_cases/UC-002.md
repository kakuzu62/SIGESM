# UC-002 - Criar Usuario

## Ator

Administrador autenticado.

## Pre-condicoes

- Administrador esta autenticado.
- Modulo Usuarios esta acessivel pelo menu lateral.

## Fluxo Principal

1. Administrador abre o modulo Usuarios.
2. Administrador aciona Novo.
3. Sistema exibe formulario de cadastro.
4. Administrador informa nome completo, login, e-mail, senha e confirmacao.
5. Sistema valida campos obrigatorios e confirmacao de senha.
6. Sistema cria `CreateUserCommand`.
7. Application valida command, value objects, duplicidade e senha.
8. Sistema persiste o usuario por Unit of Work.
9. Sistema fecha o dialogo, atualiza a listagem e exibe sucesso.

## Fluxos Alternativos

- Login duplicado: sistema mantem o dialogo aberto e informa que o login esta
  em uso.
- E-mail duplicado: sistema mantem o dialogo aberto e informa que o e-mail esta
  em uso.
- Senha fraca: sistema mantem o dialogo aberto e informa a falha da politica.
- Confirmacao divergente: sistema nao cria Command e destaca o campo.
- Cancelamento: sistema fecha o dialogo sem persistir dados.

## Pos-condicoes

- Usuario criado ativo.
- Usuario sem perfis e permissoes diretas nesta STS.
- Senha persistida apenas como hash.
- Listagem atualizada conforme consulta atual.
