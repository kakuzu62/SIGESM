# UC-004 - Ativar e Desativar Usuario

## Ator

Administrador autenticado.

## Pre-condicoes

- administrador autenticado;
- modulo Usuarios aberto;
- usuario alvo existente.

## Fluxo Principal

1. O administrador seleciona um usuario.
2. O sistema apresenta a acao de ativar ou desativar conforme o estado atual.
3. O administrador solicita a alteracao.
4. O sistema exibe mensagem de confirmacao com acao e usuario alvo.
5. O administrador confirma.
6. O sistema executa o caso de uso na camada Application.
7. O sistema persiste a mudanca e atualiza a listagem atual.

## Fluxos Alternativos

- Cancelamento: a operacao termina sem chamar Application.
- Usuario inexistente: o sistema exibe erro compreensivel.
- Auto-desativacao: o sistema rejeita a operacao.
- Mesmo estado: o sistema informa que o usuario ja esta naquele estado.

## Pos-condicoes

- o estado ativo/inativo reflete a decisao confirmada;
- senha, hash e dados cadastrais permanecem preservados;
- usuario inativo nao autentica.
