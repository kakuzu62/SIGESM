# Politica de Seguranca

## Versoes Suportadas

Enquanto o projeto estiver em fundacao inicial, apenas a branch `main` recebe
correcoes de seguranca.

## Relato de Vulnerabilidades

Nao registre vulnerabilidades sensiveis em issues publicas. Comunique o
mantenedor do repositorio com:

- descricao objetiva do problema;
- passos para reproduzir;
- impacto esperado;
- sugestao de mitigacao, quando houver.

## Diretrizes

- Nunca versionar segredos, senhas ou tokens.
- Usar `.env` somente para configuracao local.
- Manter dependencias revisadas.
- Preservar separacao entre dominio, infraestrutura e interface.
